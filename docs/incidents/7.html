Our First Kubernetes Outage

“I checked on things two hours later. The issue had not resolved itself. It became larger.”
Incident #7 at Saltside on 2017/04/26 by Adam Hawkins (SRE Team Lead)
Full report https://engineering.saltside.se/our-first-kubernetes-outage-c6b9249cfd3a
How it happened While running tests on the internal Kubernetes clusters, a small number of containers were deployed, overloading the disk IO capacity.
Architecture Multiple Kubernetes clusters for different use cases, each with Elastic Block Stores (EBS).
Technologies Amazon Elastic Compute Cloud (EC2), Amazon Elastic Block Store (EBS)
Root cause All activity on clusters exceeded available disk IO capacity (EBS IOPS).
Failure A complete shutdown of an internal cluster.
Impact Internal services provided by the cluster unavailable (Tiller, new pod creation, etc)
Mitigation Terminated three nodes and waited for new instances to join the cluster.
